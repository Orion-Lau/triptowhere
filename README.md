# 旅行目的地转盘

一个可以部署到 GitHub Pages 的旅行目的地转盘。用户转到目的地后，会把结果记入票数统计，并显示总票数。

## 本地预览

直接用浏览器打开 `index.html` 即可。未配置 Supabase 时，票数会临时存在当前浏览器的 `localStorage`，适合先看效果。

## 开启真实票数统计

GitHub Pages 是静态托管，不能自己写数据库。这里使用 Supabase 保存所有玩家的投票结果。

1. 在 [Supabase](https://supabase.com/) 创建一个项目。
2. 打开 SQL Editor，执行：

```sql
create table if not exists public.travel_wheel_votes (
  id bigint generated by default as identity primary key,
  destination text not null,
  created_at timestamptz not null default now()
);

alter table public.travel_wheel_votes enable row level security;

create policy "Anyone can add votes"
on public.travel_wheel_votes
for insert
to anon
with check (true);

create policy "Anyone can read vote totals"
on public.travel_wheel_votes
for select
to anon
using (true);
```

3. 在 Supabase 项目的 `Project Settings` -> `API` 中复制：
   - `Project URL`
   - `anon public` key
4. 打开 `app.js`，填写：

```js
const SUPABASE_URL = "你的 Project URL";
const SUPABASE_ANON_KEY = "你的 anon public key";
```

## 修改目的地

在 `app.js` 里修改 `destinations` 数组即可：

```js
const destinations = ["东京", "首尔", "清迈", "巴黎"];
```

## 发布到 GitHub Pages

1. 新建 GitHub 仓库。
2. 把本项目文件推送上去。
3. 在仓库 `Settings` -> `Pages` 中选择 `Deploy from a branch`。
4. Branch 选择 `main`，目录选择 `/root`，保存。

稍等片刻后，GitHub 会给你一个公开访问链接。
